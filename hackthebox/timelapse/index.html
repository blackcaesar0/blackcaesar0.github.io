Penetration Testing Report: Timelapse

Scope

IP Domain
10.129.227.113 timelapse.htb / dc01.timelapse.htb

Methodology

  1. Reconnaissance — enumeration of open ports, services, and domains.
  2. Exploitation — gaining initial access through identified vulnerabilities.
  3. Privilege Escalation — escalating to administrative access.
  4. Post-Exploitation — enumerating the environment and identifying misconfigurations.

Findings

Reconnaissance

Full port scan revealed:

  • DNS (53/tcp) — Active Directory domain controller.
  • Kerberos (88/tcp) — authentication.
  • LDAP (389/tcp, 636/tcp, 3268/tcp, 3269/tcp) — Active Directory LDAP.
  • SMB (445/tcp) — guest access enabled.
  • WinRM (5986/tcp) — Windows Remote Management.

SMB enumeration found guest access enabled on the Shares share. A file named winrm_backup.zip was in the dev directory.

Using netexec, identified users legacyy, svc_deploy, Administrator and groups LAPS_Readers, Domain Admins.

Exploitation

Cracking the ZIP File

winrm_backup.zip was password-protected. Used zip2john to extract the hash and cracked it with john against rockyou.txt — password: suprem*****.

Extracting the PFX File

The ZIP contained legacyy_dev_auth.pfx, also password-protected. Used pfx2john and cracked it with john — password: thugl*****.

Extracting Certificates and Keys

Decrypted the PFX with OpenSSL to extract decrypted_private.key and legacyy_dev_auth.crt.

Gaining Initial Access

Used evil-winrm with the extracted key and certificate to connect as legacyy.

Privilege Escalation

PowerShell History

The PowerShell history file (ConsoleHost_history.txt) contained credentials for svc_deploy:

  • Username: svc_deploy
  • Password: E3R$Q62^12**********

svc_deploy Group Membership

svc_deploy is a member of LAPS_Readers, allowing it to read the LAPS password for the domain controller.

Extracting the LAPS Password

Used Get-ADComputer to retrieve the LAPS password for DC01: @t%va&Bk@;*********

Administrative Access

Connected as Administrator via evil-winrm using the LAPS password.

Post-Exploitation

Misconfigurations identified:

  1. Guest access on SMB allowed unauthorized access to sensitive files.
  2. Weak passwords on the ZIP and PFX files were easily crackable.
  3. svc_deploy had excessive privileges — access to the LAPS password.

Recommendations

  • Disable guest access on SMB shares and restrict to authenticated users.
  • Enforce strong password policies for all accounts and files.
  • Restrict the LAPS_Readers group to only necessary users and log access to LAPS passwords.
  • Store PFX files in secure locations with restricted access and rotate certificates regularly.
  • Implement a policy to clear PowerShell history and limit PowerShell to authorized users.
  • Disable unnecessary services and implement network segmentation.
